# Развёртывание бота на Amvera Cloud

Инструкция по запуску бота-напоминалки на [Amvera Cloud](https://cloud.amvera.ru/).

## 1. Подготовка

### 1.1. Репозиторий на GitHub

Проект должен быть в GitHub. Если ещё не загружен:


### 1.2. Регистрация на Amvera

1. Перейдите на [cloud.amvera.ru](https://cloud.amvera.ru/)
2. Войдите или зарегистрируйтесь
3. Нажмите **«Создать проект»**

## 2. Создание проекта

1. Выберите **«Привязать репозиторий»**
2. Подключите GitHub (если ещё не подключён)
3. Выберите репозиторий с ботом
4. Укажите ветку (обычно `main`)
5. Нажмите **«Создать»**

## 3. Переменные окружения

В проекте откройте **«Переменные окружения»** и добавьте:

| Переменная | Значение | Описание |
|------------|----------|----------|
| `BOT_TOKEN` | Токен от @BotFather | Обязательно |
| `ADMIN_IDS` | `123456789,987654321` | ID админов через запятую |
| `GROUP_ID` | `-1001234567890` | ID группы (узнать: /groupid в группе) |
| `DATABASE_PATH` | `/data/bot.db` | Путь к БД (для сохранения данных) |
| `RESTAURANT_LAT` | `52.2758` | Широта ресторана (опционально) |
| `RESTAURANT_LON` | `104.2774` | Долгота ресторана (опционально) |
| `GEO_RADIUS_M` | `200` | Радиус проверки геолокации в метрах (опционально) |

**Важно:** `DATABASE_PATH=/data/bot.db` нужен, чтобы база сохранялась при перезапусках.

## 4. Сборка и запуск

1. Amvera автоматически найдёт `amvera.yaml` в корне репозитория
2. Нажмите **«Собрать»** или дождитесь автособорки после push
3. После успешной сборки нажмите **«Запустить»** или включите приложение
4. Проверьте логи во вкладке **«Лог приложения»**

## 5. Проверка работы

1. Напишите боту в Telegram команду `/start`
2. В группе выполните `/setgroup` (если GROUP_ID не задан в переменных)
3. Загрузите расписание через `/schedule` в личку боту

## 6. Обновление бота

После изменений в коде:

```bash
git add .
git commit -m "Описание изменений"
git push
```

Amvera пересоберёт и перезапустит приложение автоматически (если включён автодеплой).

## 7. Частые проблемы

### Бот не отвечает
- Проверьте `BOT_TOKEN` в переменных окружения
- Убедитесь, что приложение в статусе «Запущено»
- Посмотрите логи на наличие ошибок

### Сообщения не приходят в группу
- Выполните `/setgroup` в группе или задайте `GROUP_ID`
- В @BotFather отключите **Group Privacy** (Turn off)

### Данные не сохраняются
- Убедитесь, что задана переменная `DATABASE_PATH=/data/bot.db`
