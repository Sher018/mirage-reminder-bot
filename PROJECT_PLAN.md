# План проекта: Telegram-бот «Напоминалка отчётов»

**Ресторанный комплекс Мираж, г. Иркутск**

---

## 1. Общее описание

Бот добавлен в группу с работниками. Все подтверждения (геолокация, фото) приходят **в группу** — администратор видит всё и может контролировать.

- **Напоминания** — в группу, по расписанию
- **Подтверждения** — работники отправляют в группу (геолокация **обязательно**, фото по желанию)
- **Недельный отчёт** — бот отправляет **админу в личку** (воскресенье 00:00)

---

## 2. Алгоритм работы бота

### 2.1. Общий цикл

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         ЕЖЕДНЕВНЫЙ ЦИКЛ БОТА                            │
└─────────────────────────────────────────────────────────────────────────┘

     ЗАГРУЗКА РАСПИСАНИЯ (раз в неделю)
              │
              ▼
     Excel: Дата | День | Username | Начало смены | Конец смены
              │
              ▼
     ═══════════════════════════════════════════════════════════════════
     КАЖДЫЙ ДЕНЬ — напоминания по времени начала смен
     ═══════════════════════════════════════════════════════════════════
              │
              ▼
     Время начала смены → напоминание в ГРУППУ
     «@user1, @user2 — подтвердите присутствие: геолокация (обязательно), фото (по желанию)»
     Окно: 15 мин
              │
              ▼
     Подтверждение В ГРУППЕ:
     • Геолокация — ОБЯЗАТЕЛЬНО (проверка: расстояние до ресторана, время)
     • Фото — по желанию (если есть — проверка EXIF/времени)
              │
              ▼
     Статус: ✅ Вовремя | ⚠️ Опоздание (мин) | ❌ Не подтвердил
              │
              ▼
     ВОСКРЕСЕНЬЕ 00:00 — отчёт админу в личку
     «Кто и на сколько опоздал» + «Кто не подтвердил»
```

---

### 2.2. Детальный алгоритм

#### Шаг 1. Загрузка расписания

| Действие | Описание |
|----------|----------|
| Где | **Только в личку боту** — админ пишет боту `/schedule` и отправляет Excel |
| В группе | Админ по желанию скидывает скрин расписания (бот не парсит скрины) |
| Приём | Админ отправляет `.xlsx` в личку боту |
| Парсинг | Дата, День недели, Username (@), Начало смены, Конец смены |
| Сохранение | Данные в БД |
| Подтверждение | Бот: «Расписание загружено» |

#### Шаг 2. Напоминания в группу

Бот читает **время начала смены из Excel** — обычно 09:00, 10:00, 12:00, 15:00, но может быть 09:30, 11:00 и т.д. (всё видно в графике).

Группирует сотрудников по времени начала и отправляет напоминание в **точное время** начала смены. Окно подтверждения — 15 мин.

Пример: смена 09:30 → напоминание в 09:30, подтвердить до 09:45.

Сообщение в группу:  
«📋 Смена 09:00. @ivanov, @petrova — подтвердите присутствие: **геолокация (обязательно)**, фото по желанию. До 09:15.»

#### Шаг 3. Приём подтверждений (в группе)

| Тип | Обязательность | Проверка |
|-----|-----------------|----------|
| **Геолокация** | **ОБЯЗАТЕЛЬНО** | Расстояние ≤ 200 м, время в окне 15 мин |
| **Фото** | По желанию | EXIF или время отправки в окне |

- Подтверждение **вовремя** (в своём окне) → ✅ Вовремя
- Подтверждение **после** окна → ⚠️ Опоздание (фиксируем минуты опоздания)
- Нет подтверждения → ❌ Не подтвердил

#### Шаг 4. Опоздания для каждой смены

Для каждой смены отдельно:
- Окно 15 мин — вовремя
- После окна — опоздание, считаем минуты (например: смена 09:00, подтвердил в 09:25 → опоздание 10 мин)

#### Шаг 5. Недельный отчёт (воскресенье 00:00)

Бот отправляет **только админу в личку** (не в группу):

```
📊 Отчёт за неделю 10–16 марта

⚠️ ОПОЗДАНИЯ:
• Иванов И. (@ivanov) — 10.03, смена 09:00: опоздание 10 мин
• Петрова М. (@petrova) — 12.03, смена 10:00: опоздание 25 мин

❌ НЕ ПОДТВЕРДИЛИ:
• Сидоров А. (@sidorov) — 11.03, смена 12:00
• Козлов П. (@kozlov) — 14.03, смена 15:00
```

## 3. Формат Excel-файла (горизонтальный)

**«График работы на неделю»** — строки = сотрудники, колонки = дни:

| № | @username / ФИО | Должность | пн | вт | ср | чт | пт | сб | вс |
|---|------------------|-----------|---|----|----|----|----|----|-----|
| 1 | @esm1rnofff      | ран рей   | 10:00-00:00 | ... | ... | ... | ... | ... | ... |
| 2 | @rahmidin_5      | ран рей   | 09:00-22:00 | ... | ... | ... | ... | ... | ... |

- **Ячейка смены:** `10:00-00:00` (начало-конец)
- **Выходной:** пусто или «ВЫХОДНОЙ»
- **Дата недели:** укажите в первой строке (ДД.ММ.ГГГГ) или отправьте после файла

Напоминание — в **точное время начала смены**. Окно подтверждения — 15 мин.

---

## 4. Важные моменты

1. **Всё в группе** — напоминания и подтверждения в общей группе, админ всё видит.
2. **Геолокация обязательна** — без гео подтверждение не засчитывается.
3. **Фото по желанию** — можно отправить дополнительно.
4. **Privacy Mode** — в @BotFather отключить, чтобы бот видел сообщения в группе.
5. **Отчёт** — только админу в личку (воскресенье 00:00).
6. **Расписание** — Excel в личку боту, скрин в группу (по желанию).

## 5. Структура базы данных (SQLite)

### Таблица `schedules` (расписание из Excel)

| Поле        | Тип     | Описание                    |
|-------------|---------|-----------------------------|
| id          | INTEGER | PK                          |
| date        | DATE    | Дата смены                  |
| day_of_week | VARCHAR | День недели (Пн, Вт...)     |
| username    | VARCHAR | @username                   |
| full_name   | VARCHAR | ФИО (опционально)          |
| shift_start | TIME    | Начало смены                |
| shift_end   | TIME    | Конец смены                 |
| created_at  | DATETIME| Когда загружено             |

### Таблица `confirmations` (подтверждения)

| Поле         | Тип     | Описание                          |
|--------------|---------|-----------------------------------|
| id           | INTEGER | PK                                |
| schedule_id  | INTEGER | FK → schedules                    |
| user_id      | INTEGER | Telegram user_id                 |
| username     | VARCHAR | @username                         |
| confirmed_at | DATETIME| Когда подтвердил                 |
| status       | VARCHAR | 'on_time' \| 'late'               |
| late_minutes | INTEGER | Минуты опоздания (0 если вовремя) |
| geo_received | BOOLEAN | Геолокация получена               |
| photo_received | BOOLEAN | Фото получено (по желанию)      |
| created_at   | DATETIME|                                   |

### Таблица `work_groups` (группа для напоминаний)

| Поле      | Тип     | Описание              |
|-----------|---------|-----------------------|
| id        | INTEGER | PK                    |
| chat_id   | BIGINT  | Telegram group id     |
| created_at| DATETIME|                       |
| updated_at| DATETIME|                       |

## 6. Кнопки быстрого доступа

Чтобы не писать команды каждый раз — **кнопки**:

| Кнопка | Действие |
|--------|----------|
| 📅 Сегодня | Показать, кто сегодня работает (по сменам) |
| 📊 Статус | Показать, кто уже подтвердил, кто ещё нет |

**Где показывать:**
- Под каждым напоминанием в группе — inline-кнопки [📅 Сегодня] [📊 Статус]
- В личке админа — постоянная клавиатура с кнопками (при `/start` или `/menu`)

## 7. Конфигурация (.env)

```
BOT_TOKEN=
ADMIN_IDS=              # ID админов (получат недельный отчёт)
GROUP_ID=               # ID группы (или бот запомнит при первом напоминании)
RESTAURANT_LAT=
RESTAURANT_LON=
GEO_RADIUS_M=200
CONFIRM_WINDOW_MINUTES=15

# Недельный отчёт: воскресенье 00:00 (только админу)
WEEKLY_REPORT_DAY=0     # 0 = воскресенье
WEEKLY_REPORT_TIME=00:00
```

Время смен берётся из Excel, фиксированные REMINDER_1–4 не нужны.
