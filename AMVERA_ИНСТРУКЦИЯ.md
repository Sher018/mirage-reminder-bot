# Как привязать репозиторий к Amvera Cloud

## Способ 1: Через GitHub (рекомендуется)

Подходит, если код уже в GitHub. При каждом `git push` Amvera будет автоматически обновляться.

### Шаг 1. Создать проект на Amvera

1. Откройте [cloud.amvera.ru](https://cloud.amvera.ru/)
2. Нажмите **«Создать проект»**
3. Выберите тип **«Приложение»**
4. Введите название (например, `mirage-bot`)
5. Выберите тариф и нажмите **«Далее»**
6. Дождитесь создания проекта

### Шаг 2. Загрузить код в проект

На этапе **«Загрузка данных»** есть два варианта:

#### Вариант А: Загрузка через интерфейс

1. Перетащите папку с проектом (или все файлы) в окно загрузки
2. Убедитесь, что загружены: `main.py`, `bot/`, `requirements.txt`, `amvera.yaml`
3. Нажмите **«Завершить»**

#### Вариант Б: Привязать GitHub (автообновление)

1. Перейдите на вкладку **«Репозиторий»** в проекте
2. Выберите **GitHub** из списка
3. Создайте токен в GitHub:
   - GitHub → Settings → Developer settings → Personal access tokens
   - **Tokens (classic)** → **Generate new token**
   - Отметьте право **repo**
   - Скопируйте токен
4. Вставьте токен в поле в Amvera
5. Укажите репозиторий и ветку (например, `main`)
6. Придумайте секрет для вебхука
7. Скопируйте **URL вебхука** из Amvera
8. В GitHub: репозиторий → **Settings** → **Webhooks** → **Add webhook**
   - Payload URL: вставьте скопированный URL
   - Content type: `application/json`
   - Secret: тот же, что в Amvera
   - События: **Just the push event**
9. Нажмите **«Add webhook»**

После этого каждый `git push` в GitHub будет запускать сборку в Amvera.

## Способ 2: Push напрямую в Amvera (без GitHub)

Подходит, если Git установлен, но GitHub не используется.

### Шаг 1. Создать проект на Amvera

Как в Способе 1, шаги 1–6.

### Шаг 2. Узнать адрес репозитория Amvera

1. Откройте проект
2. Перейдите на вкладку **«Репозиторий»**
3. Скопируйте URL вида: `https://git.amvera.ru/ваш_ник/имя_проекта`

### Шаг 3. Добавить remote и отправить код

В терминале в папке проекта:

```powershell
cd d:\Проекты\RUN-RAY

# Добавить репозиторий Amvera
git remote add amvera https://git.amvera.ru/ВАШ_НИК/ИМЯ_ПРОЕКТА

# Отправить код (если ветка main)
git push amvera main:master
```

При запросе логина и пароля укажите данные от Amvera.

---

## После загрузки кода

1. **Переменные окружения** — вкладка «Переменные окружения»:
   - `BOT_TOKEN`
   - `ADMIN_IDS`
   - `GROUP_ID`
   - `DATABASE_PATH` = `/data/bot.db`

2. **Сборка** — запустится автоматически после push или загрузки файлов

3. **Запуск** — включите приложение кнопкой «Запустить»

4. **Логи** — смотрите во вкладке «Лог приложения»

---

## Если ветка называется `main`, а не `master`

Используйте:

```powershell
git push amvera main:master
```
