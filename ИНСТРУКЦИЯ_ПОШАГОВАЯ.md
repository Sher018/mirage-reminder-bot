# Пошаговая инструкция: запуск бота на сервере Amvera

Подробное руководство от нуля до работающего бота.

---

## Часть 1. Подготовка (на вашем компьютере)

### Шаг 1.1. Установка Git (если ещё не установлен)

1. Откройте [git-scm.com/download/win](https://git-scm.com/download/win)
2. Скачайте установщик для Windows
3. Запустите и пройдите установку (оставьте настройки по умолчанию)
4. **Закройте и заново откройте** PowerShell или командную строку
5. Проверьте: введите `git --version` — должна появиться версия

---

### Шаг 1.2. Регистрация на GitHub

1. Откройте [github.com](https://github.com)
2. Нажмите **Sign up** (Регистрация)
3. Заполните email, пароль, имя пользователя
4. Подтвердите email
5. Войдите в аккаунт

---

### Шаг 1.3. Создание репозитория на GitHub

1. Нажмите **+** в правом верхнем углу → **New repository**
2. **Repository name:** например `mirage-reminder-bot`
3. Выберите **Private** (приватный) или **Public** (публичный)
4. **Не ставьте** галочки на README, .gitignore, License
5. Нажмите **Create repository**
6. Скопируйте URL репозитория (например: `https://github.com/ваш_логин/mirage-reminder-bot.git`)

---

### Шаг 1.4. Загрузка кода в GitHub

Откройте PowerShell в папке проекта:

```powershell
cd d:\Проекты\RUN-RAY
```

Выполните команды по очереди:

```powershell
# 1. Инициализация Git (если ещё не сделано)
git init

# 2. Добавить все файлы
git add .

# 3. Первый коммит
git commit -m "Бот-напоминалка для Мираж"

# 4. Подключить GitHub (подставьте СВОЙ URL!)
git remote add origin https://github.com/ВАШ_ЛОГИН/mirage-reminder-bot.git

# 5. Отправить код
git branch -M main
git push -u origin main
```

При первом `git push` GitHub может попросить войти — используйте логин и пароль (или токен).

## Часть 2. Регистрация и создание проекта на Amvera

### Шаг 2.1. Регистрация на Amvera

1. Откройте [cloud.amvera.ru](https://cloud.amvera.ru/)
2. Нажмите **«Войти»** или **«Регистрация»**
3. Зарегистрируйтесь (email, пароль) или войдите
4. Подтвердите email, если потребуется

---

### Шаг 2.2. Создание проекта

1. На главной странице нажмите **«Создать проект»**
2. Выберите тип: **«Приложение»**
3. Введите название: например `mirage-bot`
4. Выберите тарифный план (есть бесплатный)
5. Нажмите **«Далее»**
6. Дождитесь создания проекта (появится экран «Загрузка данных»)

---

### Шаг 2.3. Загрузка кода — Вариант А (через интерфейс)

1. На экране **«Загрузка данных»** найдите область перетаскивания файлов
2. Откройте папку `d:\Проекты\RUN-RAY` в проводнике
3. Выделите **все файлы и папки**, кроме:
   - `.env` (не загружать — там секреты!)
   - `.git` (скрытая папка)
   - `bot.db`
   - `debug_*.txt`
4. Перетащите выделенное в окно Amvera
5. Дождитесь загрузки
6. Проверьте, что есть: `main.py`, папка `bot`, `requirements.txt`, `amvera.yaml`
7. Нажмите **«Завершить»** или **«Далее»**

---

### Шаг 2.4. Загрузка кода — Вариант Б (через GitHub)

1. На экране создания проекта выберите **«Привязать репозиторий»** (если есть такой вариант)
2. Или после создания проекта перейдите на вкладку **«Репозиторий»**
3. Выберите **GitHub** из списка git-сервисов
4. **Создайте токен в GitHub:**
   - Откройте [github.com/settings/tokens](https://github.com/settings/tokens)
   - **Tokens (classic)** → **Generate new token (classic)**
   - Note: `Amvera`
   - Expiration: `No expiration` или 90 дней
   - Отметьте галочку **repo**
   - Нажмите **Generate token**
   - **Скопируйте токен** (он показывается один раз!)
5. Вставьте токен в поле в Amvera
6. Выберите ваш репозиторий и ветку `main`
7. Придумайте секрет (любая строка, например `mysecret123`)
8. Скопируйте **URL вебхука** из Amvera
9. **Настройте вебхук в GitHub:**
   - Откройте ваш репозиторий на GitHub
   - **Settings** → **Webhooks** → **Add webhook**
   - Payload URL: вставьте скопированный URL
   - Content type: `application/json`
   - Secret: тот же, что в Amvera
   - Which events: **Just the push event**
   - Нажмите **Add webhook**
10. В Amvera нажмите **«Применить»**
11. Сделайте любой `git push` в GitHub — Amvera подтянет код

## Часть 3. Настройка переменных окружения

### Шаг 3.1. Открыть настройки переменных

1. В проекте Amvera откройте вкладку **«Переменные окружения»** (или «Конфигурация» → «Переменные»)
2. Нажмите **«Добавить»** или **«Добавить переменную»**

---

### Шаг 3.2. Добавить переменные по одной

Добавьте каждую переменную (имя и значение):

| № | Имя | Значение | Где взять |
|---|-----|----------|-----------|
| 1 | `BOT_TOKEN` | Токен бота | @BotFather в Telegram → /mybots → выбрать бота → API Token |
| 2 | `ADMIN_IDS` | Ваш Telegram ID | Написать @userinfobot в Telegram, скопировать ID |
| 3 | `GROUP_ID` | ID группы | В группе написать боту `/groupid` (или добавить позже) |
| 4 | `DATABASE_PATH` | `/data/bot.db` | Писать точно так |
| 5 | `RESTAURANT_LAT` | `52.230755` | Координаты ресторана (опционально) |
| 6 | `RESTAURANT_LON` | `104.314715` | Координаты ресторана (опционально) |
| 7 | `GEO_RADIUS_M` | `300` | Радиус в метрах (опционально) |

**Важно:** `DATABASE_PATH` и `BOT_TOKEN` — обязательны. Остальные можно добавить позже.

---

### Шаг 3.3. Сохранить переменные

Нажмите **«Сохранить»** или **«Применить»**.

---

## Часть 4. Сборка и запуск

### Шаг 4.1. Запустить сборку

1. Если сборка не началась автоматически — нажмите **«Собрать»** или **«Сборка»**
2. Откройте вкладку **«Лог сборки»** или **«Сборка»**
3. Дождитесь окончания (статус «Успешно» или «Сборка завершена»)
4. Если ошибка — посмотрите лог, проверьте наличие `requirements.txt` и `amvera.yaml`

---

### Шаг 4.2. Запустить приложение

1. Найдите кнопку **«Запустить»** или переключатель «Включить/Выключить»
2. Включите приложение
3. Статус должен смениться на **«Приложение запущено»** или «Выполняется»
4. Откройте вкладку **«Лог приложения»** — должны появиться строки вида «Бот запущен»

---

## Часть 5. Проверка работы

### Шаг 5.1. Проверить бота в Telegram

1. Откройте Telegram
2. Найдите вашего бота
3. Напишите `/start`
4. Бот должен ответить приветствием

---

### Шаг 5.2. Настроить группу (если ещё не сделано)

1. Добавьте бота в группу с работниками
2. Сделайте бота администратором группы
3. В группе напишите `/setgroup`
4. Бот ответит «Группа сохранена»
5. Если `GROUP_ID` не был задан в переменных — скопируйте ID из ответа бота и добавьте в переменные Amvera, затем перезапустите приложение

---

### Шаг 5.3. Отключить Group Privacy в BotFather

1. Откройте @BotFather в Telegram
2. Отправьте `/mybots`
3. Выберите вашего бота
4. **Bot Settings** → **Group Privacy** → **Turn off**
5. Иначе бот не будет получать сообщения и геолокацию от участников группы

---

### Шаг 5.4. Загрузить расписание

1. Напишите боту **в личку** команду `/schedule`
2. Отправьте Excel-файл с расписанием
3. Если бот попросит дату — отправьте дату понедельника (например `10.03.2025`)
4. Бот ответит «Расписание загружено»

---

Когда вы изменили код на компьютере:


## Часть 7. Решение проблем

### Бот не отвечает в Telegram

- Проверьте `BOT_TOKEN` в переменных окружения
- Убедитесь, что приложение в статусе «Запущено»
- Откройте «Лог приложения» — нет ли ошибок

### Сообщения не приходят в группу

- Выполните `/setgroup` в группе
- Добавьте `GROUP_ID` в переменные (из ответа `/groupid`)
- В @BotFather отключите Group Privacy

### Расписание не сохраняется / данные пропадают

- Добавьте переменную `DATABASE_PATH` со значением `/data/bot.db`
- Перезапустите приложение

### Ошибка при сборке

- Проверьте, что в корне проекта есть `amvera.yaml` и `requirements.txt`
- Посмотрите «Лог сборки» — на какой строке ошибка
- Убедитесь, что все зависимости указаны в `requirements.txt`

## Как протестировать напоминания

Чтобы проверить, что напоминания за 5 минут до смены работают и в следующих напоминаниях не показываются уже отметившиеся:

1. **Загрузите расписание на сегодня**  
   Через кнопку «Расписание» или команду загрузки Excel на день с тремя сменами (например 09:00, 12:00, 15:00).

2. **Проверка первого напоминания (как в 8:55)**  
   - В личку боту отправьте: `/test_reminder`  
   - В группу должно прийти сообщение «Сегодня работают» со всеми сменами и всеми работниками.  
   - Это то же сообщение, что бот шлёт в 8:55.

3. **Проверка фильтра «кто уже отметился» (как в 11:55 и 14:55)**  
   - Пусть один работник с первой смены (9:00) отметится: ответит на сообщение бота геолокацией.  
   - В личку боту отправьте: `/test_reminder 12:00`  
   - В группу придёт напоминание «как перед сменой 12:00»: в блоке **09:00** должны быть только те, кто ещё не отметился (или «— все отметились», если все отметились). Смены 12:00 и 15:00 — полным списком.

4. **Проверка третьей смены**  
   - После того как часть людей со смен 9:00 и 12:00 отметится, отправьте: `/test_reminder 15:00`  
   - В сообщении для 09:00 и 12:00 должны быть только не отметившиеся (или «— все отметились»).

**Важно:** `/test_reminder` и `/test_reminder 12:00` только отправляют сообщение в группу; они не ждут реального времени 8:55 или 11:55. Реальные напоминания по времени продолжает отправлять планировщик (каждую минуту проверяет: пора ли отправить за 5 мин до смены).
